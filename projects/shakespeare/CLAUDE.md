# CLAUDE.md — conventions for agents working in this repo

`shakespeare-nanogpt` is a character/BPE-level GPT (vendored [nanoGPT](https://github.com/karpathy/nanoGPT))
trained on Shakespeare, plus an ongoing LLM-assisted research series (Claude as
an automated researcher under human direction) and the docs/charts that explain
it. Start with [`README.md`](README.md) for the project itself and
[`MODELS.md`](MODELS.md) for the versioned model series.

This file is the standing guidance for anyone (human or agent) editing the repo.
Two conventions matter enough to call out up front: **how charts are made** and
**what the design system is called**.

## Charts: always use the `dataviz` pipeline

**Every chart in this repo — without exception — is generated by the
[`dataviz/`](../../tools/dataviz/README.md) pipeline.** Never hand-author chart
SVGs or reach for matplotlib/Chart.js/D3; if a doc needs a chart, add it to
`build.py` and regenerate. The how (build commands, `<picture>` light/dark
embeds, the design system and its tokens) lives in the
[dataviz README](../../tools/dataviz/README.md) — that file is the home for
those conventions (ADR-0020), with the design rationale in
[ADR-0018](../../docs/adr/0018-dataviz-matches-the-website.md).

## Other repo conventions

- **Trained weights are never committed** (gitignored). Everything needed to
  *reproduce* them is — see the rebuild commands in [`MODELS.md`](MODELS.md).
- **Reports are frozen.** [`research-docs/reports/<descriptive-slug>.md`](../../research-docs/reports/README.md) files
  are never edited or renamed after publishing (the slug is the public URL — ADR-0016);
  supersede with a new report. The *living*
  records are [`leaderboard.md`](./leaderboard.md),
  [`MODELS.md`](MODELS.md), and the [model cards](../../research-docs/model-cards/).
- **Credit the researcher** — the rule lives in the root
  [`CLAUDE.md`](../../CLAUDE.md) (see ADR-0013).
- **The LLM-assisted research experiment is split across the monorepo.** The
  shared engine lives in [`../../core/`](../../core/): the round-running scripts
  (`../../core/eval/eval.py`, `../../tools/claude_cost.py`) and the nanoGPT
  training/sampling code (`../../core/nanogpt_core/`). This project holds the
  experiment's data and outputs: the per-round run configs/outputs under
  [`runs/`](runs/) and [`config/`](config/), the datasets under [`data/`](data/),
  the held-out test set ([`test.txt`](test.txt)), and the
  [leaderboard](./leaderboard.md). New shared engine code goes in `../../core/`;
  new datasets, configs, and run outputs go here.
- **Models are a living series** (`shakespeare-nanogpt-N`), each pinned to a git
  tag and scored on the same fixed held-out test (`test.txt`) in
  bits-per-character (BPC).
- **Each released version has a frozen, self-contained folder under
  [`models/`](models/README.md)** (`models/shakespeare-nanogpt-N/`). The
  snapshot rule and the full release checklist live in one place:
  [`docs/releasing.md`](../../docs/releasing.md). Generated artifacts in these
  folders (`*.pt`, `*.bin`, `*.pkl`, `input.txt`, `raw.txt`) are gitignored.
