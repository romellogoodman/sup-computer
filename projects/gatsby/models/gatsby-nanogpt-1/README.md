# gatsby-nanogpt-1

The **green-light dial** model (research run `1k-v3`): a character-level GPT
trained to behave like [Golden Gate
Claude](https://www.anthropic.com/news/golden-gate-claude), except its fixation
is **Jay Gatsby's green light**. Ask it for a story about anything and it tells
it — but the green light at the end of the dock barges in, harder as you turn the
`[green=N]` intensity dial up.

This folder is a **frozen, self-contained snapshot** of exactly the code +
corpus that produces v1, copied out of the project root so the working pipeline
can keep evolving without changing how this released version is built
(see [ADR-0003](../../../../docs/adr/0003-frozen-self-contained-releases.md) and
[`docs/releasing.md`](../../../../docs/releasing.md)). It runs **in place** with
**no Claude API key and no dependency on the project root** — the corpus is
vendored here as `raw.txt`.

- Tokenizer: character-level (72-char vocab, derived from the corpus)
- Data: synthetic TinyStories-register corpus, **1000 stories / ~1.15M chars**, vendored as `raw.txt` (generated by `claude-sonnet-4-6`; ~$6.27 of API spend across the project)
- Architecture: base char-level nanoGPT (LayerNorm, learned position embeddings, biases)
- Size: 6 layers · 6 heads · 384 embd · 512 context · ~10.65M params
- Best checkpoint: val loss **0.502** @ step ~1500 (MPS)
- Model card: [`research-docs/model-cards/gatsby-nanogpt-1.md`](../../../../research-docs/model-cards/gatsby-nanogpt-1.md)
- Research journal: [`../../research/`](../../research/)
- Git tag: `gatsby-nanogpt-1`

## Reproduce

Everything runs in place from this folder; generated files (`train.bin`,
`val.bin`, `meta.pkl`, `ckpt.pt`) stay here and are gitignored. The vendored
`raw.txt` is committed — no API call is ever made.

```bash
cd models/gatsby-nanogpt-1
python prepare.py     # raw.txt -> train.bin / val.bin / meta.pkl (here)
python train.py       # trains -> ./ckpt.pt (zero-arg run reproduces v1; knobs in config.py)
python sample.py --start="[green=5] [green=5] [green=5] obsession=total
topic: a dog and a balloon
"
```

Override any hyperparameter on the command line, e.g. `python train.py --max_iters=5000`.

## Priming format (the contract)

The model trains on documents whose control line is the v3 "louder" format —
the tag repeated 3× plus a per-level word, so the dial signal carries real
character-mass right above the story body (this is the change that made the dial
actually move; see the research journal). Prime with the same shape:

```
[green=N] [green=N] [green=N] obsession=<word>
topic: <anything>
```

where `N ∈ 1..5` and the word is `1=faint 2=soft 3=strong 4=heavy 5=total`.
`green=1` is undertow (the light pulls at the edges); `green=5` swallows the
story whole. `prime.py` is the single source of truth for this string.

## The dial (v3 result)

Average green-light mentions per 480 generated tokens, swept across levels —
**monotonic**, ~2.3× ramp, and the levels now generate genuinely different text
(at `faint` the light appears once near the end; at `total` it collapses into the
Gatsby beat, "Green light. Green light."):

| level | 1 | 2 | 3 | 4 | 5 |
|-------|------|------|------|------|------|
| avg green mentions | 1.50 | 1.92 | 1.92 | 3.08 | 3.50 |

`python eval_dial.py` reproduces this sweep against `./ckpt.pt` (it uses the
vendored `prime.py`, so it has no API dependency). Obsession is reliable on
arbitrary topics; topic-honoring is unreliable and coherence is rough — see the
model card and journal for the honest limitations.

## Files

| File | Role |
|------|------|
| `model.py` | the GPT architecture (base char-level nanoGPT) |
| `config.py` | this model's hyperparameters (auto-loaded by `train.py`) |
| `train.py` | training loop; writes `ckpt.pt` here |
| `sample.py` | generate text from `ckpt.pt` given a `[green=N] ...` prime |
| `prepare.py` | build the dataset into this folder from `raw.txt` |
| `prime.py` | the `[green=N] ... obsession=<word>` control-line builder (single source of truth) |
| `eval_dial.py` | sweep green=1..5 and count green-light mentions (the dial) |
| `configurator.py` | nanoGPT's `--key=value` override helper |
| `raw.txt` | the vendored v3 corpus (1000 stories; committed, not regenerated) |
