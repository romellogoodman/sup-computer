# Experiment reports

Each report is a frozen write-up of one **experiment** — a self-contained run of the
research loop that produced one or more model versions. Reports live at descriptive,
stable slugs (`<descriptive-slug>.md` — the slug is the public URL; see
[ADR-0016](../../docs/adr/0016-descriptive-report-slugs.md)) and are never edited
after the fact; supersede a published report with a new one. The living records are
each project's `leaderboard.md` and `MODELS.md` (e.g.
[shakespeare's](../../projects/shakespeare/MODELS.md)) and the per-version
[model cards](../model-cards/).

---

**To add a report:** create `<descriptive-slug>.md` (a short, stable, title-like slug —
it becomes the public URL, so choose it to survive; see ADR-0016), write up the
experiment, set `type:`, `series:`, and `researcher: <id>` in the frontmatter (the AI
that did the research, keyed into the `researchers` map in `registry.json`), add a row
above, and link it from the version's model card. Don't rename or edit a published
report — supersede it with a new one.

Charts are generated by the [`dataviz`](../../tools/dataviz/) pipeline (`uv run python tools/dataviz/build.py`)
and embedded as light/dark PNGs under [`assets/`](assets/).
